# Правила регулярного импорта слов из Google Docs

Эти правила применяются к каждому обновлению слов Насти и Лёни (вечером в четверг и вечером в воскресенье). Импорт не должен просто копировать строки из документа: сначала записи приводятся к учебному виду, затем проверяются и только после этого добавляются в словарь профиля.

## 1. Источник и профиль

- Документ Насти добавляется только в словарь Насти; документ Лёни — только в словарь Лёни.
- Каждая новая запись получает `source` (документ и дата занятия/импорта).
- Недельный набор хранится отдельно в `NASTYA_WEEKLY_LESSONS` и
  `LENYA_WEEKLY_LESSONS`. Нельзя ставить `isWeekly:true` всем записям файла
  импорта.
- При каждом запуске недельный набор полностью пересобирается по последним
  датированным занятиям: обычно это два последних занятия или последние
  7–10 календарных дней. Буквальные страницы Google Docs не использовать как
  единственный ориентир: их границы нестабильны.
- Старые карточки не удаляются и остаются в тематических категориях, но
  исключаются из недельного набора. Их `id`, прогресс и история повторений
  сохраняются.
- Уже импортированные строки не добавляются повторно. Дубликат определяется после нормализации иврита, пробелов, кавычек, дефисов и огласовок.
- Если запись уже есть, обновляются только явно исправленные поля; прогресс пользователя и историю повторений нельзя сбрасывать.

## 2. Нормализация карточки

1. Оставить на лицевой стороне только изучаемое слово или короткую устойчивую фразу на иврите.
2. На оборотной стороне хранить транскрипцию, перевод и один короткий естественный пример.
   Транскрипция обязательна для каждой карточки, включая все формы глагола.
   Сначала использовать проверенную транскрипцию из конспекта; если её нет —
   создать локально, проверить неоднозначные чтения по смыслу и только затем
   публиковать. Автоматически созданную транскрипцию нельзя показывать на
   лицевой стороне: она появляется только после переворота карточки.
3. Длинные предложения, диалоги и пояснения не делать отдельными словами: сохранить их как `example` у ближайшего слова или фразы.
4. Исправить порядок слов на естественный израильский иврит; не переносить в карточку буквальный порядок из русского.
5. Переводить в соответствующем времени и лице. Не смешивать инфинитив, прошедшее и настоящее время.
6. Убрать лишние знаки, черновые пометки, несколько вариантов перевода в одной строке и технический текст.
7. Для существительных и прилагательных сохранять род/число в пояснении, если это важно для употребления.

## 3. Категоризация

Использовать только эти десять тем и выбирать одну основную тему на карточку:

`Глаголы`, `Грамматика`, `Повседневное`, `Фразы`, `Еда`, `Работа`, `Здоровье`, `Передвижение`, `Идиомы и сленг`, `Песни`.

Приоритет выбора: явная тема урока → смысл и ситуация употребления → устойчивый контекст примера. Если слово подходит к нескольким темам, выбрать тему, в которой пользователь скорее всего будет его искать. Не создавать новые категории и не оставлять технические названия вроде «лексика» или «слова Лёни».

## 4. Особые правила для глаголов

- Любой инфинитив, личная форма или прошедшая/настоящая форма сначала сопоставляется с одной уникальной глагольной семьёй.
- Для каждой семьи создать до трёх карточек: инфинитив (`ל...`), `он` в прошедшем времени и `я` (м.р.) в настоящем времени.
- Если инфинитив входит в недельный набор, недельными считаются все доступные
  учебные формы этой семьи. Сам инфинитив-дубль из старого словаря в очередь
  не добавляется.
- На каждой форме указать правильный перевод именно этой формы; например, настоящее `אני קונה` — «я покупаю», а не «купить».
- Если форма уже существует в общей базе, связать её с существующей семьёй, а не создавать новую.
- В карточках серии хранить `verbFamilyId` и `formLabel` (`Инфинитив`, `Он · прошедшее`, `Я · настоящее · м.р.`).
- Счётчик темы считает уникальные глагольные семьи, а глагол считается выученным только после отметки всех доступных форм семьи.
- Неполные или сомнительные формы отправлять в список проверки, не публиковать молча.

## 5. Контроль перед публикацией

Перед каждым обновлением проверить:

- нет ли дублей и пустых ивритских полей;
- у каждой карточки есть перевод и понятная тема;
- у каждой карточки есть непустая транскрипция;
- длинные предложения вынесены в примеры;
- порядок слов и соответствие времени проверены;
- глагольные формы связаны с правильными семьями;
- новые слова попали только в нужный профиль;
- недельный набор содержит только карточки последних датированных занятий;
- ни одна старая запись не стала недельной только потому, что находится в
  `doc-imports.js`;
- старые `id` и прогресс не изменились.

После проверки обновить словарь и `transcriptions.js`, пересчитать темы и
уникальные глагольные семьи. Затем обязательно выполнить `npm test`.
Публиковать приложение можно только после сообщения
`Проверка пройдена`: эта проверка автоматически ловит пустые поля, точные
дубли, неизвестные категории, отсутствующие транскрипции, повторяющиеся
идентификаторы и неполные формы глаголов.
